## 2024-05-19 - Optimizing Database Queries in add_test loop
**Learning:** The `add_test` view iterates over `test_names` submitted by the user. Inside this loop, it executes `BloodTestInfo.objects.filter(test_name=test_name).first()` to fetch the reference range information for each test before creating the `BloodTest` record. If a user submits a form with many test types at once, this results in an N+1 query problem against the `BloodTestInfo` table.
**Action:** Before the loop, fetch all the required `BloodTestInfo` records in a single query using `.filter(test_name__in=test_names)` and store them in a dictionary keyed by `test_name`. Inside the loop, do a fast O(1) dictionary lookup instead of hitting the database. This significantly improves performance when multiple tests are added simultaneously without changing any core logic or error handling behavior.
